How to Deploy Your Dashboard to GitHub Pages
=============================================

1. Create a GitHub account (if you don’t have one):
   https://github.com/join

2. Create a new repository:
   - Go to https://github.com/new
   - Name it something like `rajan-profile-dashboard`
   - Set it to Public
   - Do NOT initialize with a README (optional)

3. Upload your files:
   - On your repo page, click “Add file” > “Upload files”
   - Drag and drop all files and folders from your `Rajan_Profile_Dashboard_v1.0` directory (including index.html, css, pages, etc.)
   - Click “Commit changes” to upload.

4. Enable GitHub Pages:
   - Go to your repo’s “Settings” > “Pages” (or “Code and automation” > “Pages”)
   - Under “Source”, select “Deploy from a branch”
   - Select the `main` branch and `/ (root)` folder, then click “Save”

5. Get your site URL:
   - After a minute, your site will be live at:
     https://YOUR_GITHUB_USERNAME.github.io/rajan-profile-dashboard/
   - (Replace `YOUR_GITHUB_USERNAME` with your GitHub username.)

6. Access your dashboard:
   - Open the above URL in your browser. All navigation and pages should work.

Notes:
- All files and folders (css, pages, etc.) must be uploaded as they are in your local project.
- If you update your site, just upload/commit the new files and GitHub Pages will update automatically.
- For more help: https://pages.github.com/
